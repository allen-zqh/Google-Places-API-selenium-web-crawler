# A Web Crawler for Appendix A

This repository provides the web data collection workflow used in Appendix A of the manuscript:

**"University as a knowledge source of innovation: A spatial analysis of the impact on local science-based startup creation"**

The purpose of this repository is to document the procedures used to collect firm homepage URLs and crawl webpage contents for identifying science-based firms. The repository is provided for transparency and methodological documentation. It does not contain the proprietary firm-level data, Google Places API keys, METI survey data, crawled webpage text, or the final empirical dataset used in the manuscript.

## Workflow overview

The web data collection process consists of two main steps.

### 1. Retrieving firm homepage URLs

Notebook:

```text
googleAPI_name_location_url.ipynb
```

This notebook uses firm names and geocoded firm locations to query the Google Places API. The procedure is designed to retrieve firm-specific information, especially homepage URLs.

The logic follows the process described in Appendix A:

1. Use the geocoded firm address as the center of a search area.
2. Search for the firm name within a specified radius.
3. Retrieve the Google Maps `Place_ID` for the matched entity.
4. Use the `Place_ID` to request detailed information, including the homepage URL.
5. Save the matched firm-level URL information for subsequent webpage crawling.

The use of `Place_ID` helps reduce ambiguity when multiple firms are located in the same building or when firm names are similar.

### 2. Crawling firm homepage contents

Notebook:

```text
web_crawler.ipynb
```

This notebook crawls textual content from firm homepages and first-level subpages.

The crawler is designed to:

1. Visit the homepage URL obtained from the Google Places API.
2. Detect clickable links and first-level subpages.
3. Exclude external links based on domain similarity.
4. Use Selenium to render pages when necessary.
5. Extract webpage text for downstream text classification.
6. Remove non-webpage files, such as `.pdf`, `.doc`, `.docx`, `.xls`, and `.xlsx`, when the purpose is to analyze webpage text rather than attached documents.

The crawled webpage contents are used as input for the science-based firm classification process described in Appendix A.

## Notes on domain filtering

The crawler excludes external links using domain similarity. In the manuscript workflow, a threshold of approximately `0.75` was used to retain subpages belonging to the same firm website while excluding unrelated external domains.

Users may adjust this threshold depending on the structure of firm websites and the quality of retrieved links.

## Notes on responsible crawling

Users should apply responsible crawling practices, including:

- checking website terms of use;
- respecting `robots.txt` where applicable;
- limiting request frequency;
- setting reasonable timeouts;
- avoiding excessive parallel requests;
- storing only information needed for research purposes.

This repository is intended for research documentation and should not be used for high-frequency or intrusive crawling.

## Relation to the manuscript

In the manuscript, the collected homepage contents are used to identify science-based firms. Specifically, Appendix A describes a workflow in which firm homepages and subpages are crawled, preprocessed, and used as textual input for a supervised classification model.

The METI university-startup list is used as a high-confidence positive source for model training and validation. The final empirical sample is broader than formal university spin-offs and is designed to capture science-based firms whose activities rely on scientific or technological knowledge.

This repository documents the webpage URL retrieval and crawling components of that workflow. It does not include the subsequent machine-learning classification model, the METI labels, or the regression analysis.
